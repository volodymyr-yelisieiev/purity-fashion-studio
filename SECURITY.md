# Security policy

## Supported version

Security fixes are applied to the current production version on `main`.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
vulnerability reporting flow under **Security → Advisories → Report a
vulnerability**.

Include the affected route or component, reproduction steps, impact, and any
known mitigation. Reports are acknowledged as soon as practical and handled
privately until a fix is available.
